## Strategy Description

### Multi-Factor Momentum Strategy

Three market signal dimensions are combined into a composite score to determine buy/sell timing:

| Factor | Weight | Calculation |
|--------|--------|-------------|
| Price Momentum | 40% | Return over the past 20 candles; positive = uptrend |
| Volume Factor | 30% | Current volume / 20-bar average volume; volume-price alignment adds score |
| Volatility Factor | 30% | Standard deviation of returns; high volatility reduces signal strength |

**Trading Rules:**
- Composite score > **0.4** → Buy (deploy 95% of available cash)
- Composite score < **-0.4** → Sell (liquidate full position)
- Otherwise → Hold

**Signal Execution (no look-ahead bias):**
- Factor score is computed at the close of bar *i*
- Actual fill occurs at the **open price** of bar *i+1* (with 0.02% slippage)
- Ensures the strategy never uses future information

## Quick Start

### 1. Install Dependencies

```bash
pip install -r requirements.txt
```

### 2. Fetch Historical Data (required for first run)

```bash
python main.py --fetch
```

### 3. Run Backtest

```bash
python main.py --backtest
```

After the backtest completes, results are automatically written to `backtest_cache.json` (do not edit manually).

### 4. Launch GUI Dashboard

```bash
streamlit run gui/dashboard.py
# Open browser at http://localhost:8501
```

### 5. Simulated Live Trading

```bash
python main.py --live --rounds 20
```

### 6. Full Pipeline

```bash
# With interactive pause (press Enter after backtest to continue Paper Trading)
python main.py

# Fully automated mode (no manual Enter required, suitable for scripting)
python main.py --no-pause
```

## Risk Control Rules

- Commission rate: 0.1% (Binance spot standard)
- Slippage estimate: 0.02% (conservative estimate for high-liquidity spot)
- Position sizing: 95% of available cash per buy order
- Max drawdown stop: trading halts automatically when account drops more than **15%** from peak

---

## TCA Notes

This project implements a **basic TCA (Transaction Cost Analysis)**, including:

| Cost Item | Calculation | Notes |
|-----------|-------------|-------|
| Commission | Trade value × 0.10% | Binance spot rate |
| Slippage | Trade value × 0.02% | Empirical ratio estimate, not order-book driven |

**Limitations:**
- Slippage is based on an empirical ratio, not real fill data or a market impact model (e.g., Almgren-Chriss)
- Does not simulate partial fills, cancellations, or full order lifecycle
- Suitable for a course prototype, not a production-grade execution system
